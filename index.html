How it works

1. Send transactional email

POST messages with project-scoped tokens and strict sender-domain enforcement.

2. Receive inbound as events

Provision inboxes and receive signed webhook payloads, attachments included via signed URLs.

3. Operate via control-plane API

Guardrails, telemetry, cleanup, and security actions available under admin API routes.

Strong API story

If it exists in the product, it should be available through API. Dashboard is optional.

  • Project-scoped and admin-scoped auth
  • Idempotency support for mutating admin control-plane endpoints
  • Machine-readable OpenAPI contract for admin routes

What Linje is not

Included

  • Transactional outbound email
  • Inbound webhooks with retries
  • Operational control-plane APIs

Explicitly excluded

  • Campaigns and list management
  • Template editors and marketing automation
  • Open/click analytics suite
