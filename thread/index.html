Concept 03 / Manual

API for transactional email. Webhooks for incoming email.

Linje is deliberately small. It helps software send operational email and turn inbound email into something your application can consume.

Two jobs

  1. Send product email. Receipts, password resets, invites, account notifications.
  2. Receive product email. Provision inboxes and receive signed webhook events with attachments.

Example requests

Send /v1/messages 
curl -X POST /v1/messages \
  -H 'content-type: application/json' \
  -d '{
    "from": "no-reply@tx.example.com",
    "to": ["you@example.com"],
    "subject": "Welcome"
  }'
Receive /v1/inboxes + webhook 
{
  "from": "user@example.com",
  "to": "support@in.example.com",
  "subject": "Reply",
  "attachments": [...]
}
